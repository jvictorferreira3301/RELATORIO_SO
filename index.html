Capítulo 1
Introdução

O gerenciamento eficiente da memória é uma parte crucial do desempenho de sistemas operacionais, especialmente em ambientes com recursos limitados. A memória swap, ou memória virtual, desempenha um papel fundamental ao oferecer recursos adicionais para armazenamento temporário quando a memória RAM atinge seus limites.

Este estudo se concentra em explorar o impacto do uso de swap em aplicações que demandam uma quantidade significativa de memória RAM. Em ambientes onde a memória física disponível é restrita, a utilização eficaz da memória swap pode influenciar diretamente o desempenho e a responsividade do sistema.

Capítulo 2
Objetivos

O presente estudo tem como objetivos:

  • Analisar o impacto do uso de memória swap em aplicações que demandam uma quantidade significativa de memória RAM.
  • Investigar a medida em que a adição de memória swap é ou deixa de ser eficiente para o sistema testando múltiplos cenários de swap.

Capítulo 3
Desenvolvimento

Neste capítulo, é descrito o desenvolvimento da aplicação em python feita para medir a eficácia do uso de swap. O objetivo principal desta aplicação é simular um cenário de carga de trabalho intensiva em RAM, onde a demanda por recursos de memória é significativamente elevada e posteriormente avaliar a execução da aplicação nos seguintes 3 cenários distintos de swap:

1.
Sem Swap (0gb): A execução sem o uso de swap, onde toda a carga é tratada pela RAM principal.
2.
Swap Igual à RAM (8gb): A execução com uma configuração de swap equivalente à capacidade da RAM disponível.
3.
Swap duas vezes maior que a RAM (16gb): A execução com uma configuração de swap que é o dobro da capacidade da RAM disponível.

3.2 Obtenção dos Dados para Análise

Para coletar os dados do uso do swap nos 3 cenários descritos anteriormente (3), foi feito um script em bash que é mostrado na figura 2. O script em Bash realiza uma série de testes de desempenho relacionados à utilização de memória swap em um ambiente Linux.

Abaixo está uma explicação detalhada das principais etapas do script:

1.
Teste sem memória de Swap:
  • Verifica se há alguma partição de swap ativa e a desativa, se existir.
  • Inicia a coleta de estatísticas do sistema usando o comando vmstat 1 60 e redireciona a saída para swap0gb.txt.
  • Inicia a execução do código em Python desenvolvido memoria_swap.py (2) após uma pausa de 10 segundos.
  • Aguarda a conclusão do memoria_swap.py (2) antes de encerrar a coleta de estatísticas.
2.
Teste com 8GB de Swap:
  • Cria um arquivo de swap (/swapfile) com tamanho de 8GB.
  • Atribui permissões adequadas ao arquivo de swap.
  • Formata o arquivo de swap e ativa o swap.
  • Inicia a coleta de estatísticas do sistema usando vmstat 1 60 e redireciona a saída para swap8gb.txt.
  • Inicia a execução do memoria_swap.py (2) após uma pausa de 10 segundos.
  • Aguarda a conclusão do memoria_swap.py (2) antes de encerrar a coleta de estatísticas.
3.
Teste com 16GB de Swap:
  • Desativa e remove o swap existente, se houver.
  • Cria um novo arquivo de swap de 16GB.
  • Atribui permissões adequadas ao novo arquivo de swap.
  • Formata o novo arquivo de swap e ativa o swap.
  • Inicia a coleta de estatísticas do sistema usando vmstat 1 60 e redireciona a saída para swap16gb.txt.
  • Inicia a execução do memoria_swap.py (2) após uma pausa de 10 segundos.
  • Aguarda a conclusão do memoria_swap.py (2) antes de encerrar a coleta de estatísticas.

Ao final de cada teste, o script ajusta as permissões dos arquivos de saída (swap0gb.txt, swap8gb.txt, swap16gb.txt) para permitir o acesso. Os resultados desses testes fornecem informações sobre o impacto da configuração da memória de troca no desempenho do sistema em um ambiente Linux.

Como descrito, nesses arquivos de texto citados são gravadas estatísticas oriundas do vmstat, porém nem todas as estatísticas gravadas são interessantes, então, utilizando a biblioteca pandas do Python, foi feito um tratamento nos dados para que fossem mostradas, em um dataframe, somente as seguintes informações:

  • swpd (Swap Used): Indica a quantidade de espaço de swap que está atualmente em uso no sistema. Valores não nulos indicam que o sistema está fazendo uso do swap.
  • si (Swap In): Indica a quantidade de dados sendo movida do disco para a memória swap (swap in). Um aumento em ‘si‘ significa que o sistema está transferindo dados do swap de volta para a memória principal.
  • so (Swap Out): Indica a quantidade de dados sendo movida da memória para o disco swap (swap out). Um aumento em ‘so‘ significa que o sistema está transferindo dados da memória principal para o swap.
  • us (CPU User Time): Porcentagem de tempo da CPU gasto na execução de código de usuário. Se o swap estiver sendo usado intensivamente, pode haver um aumento na carga da CPU.
  • sy (CPU System Time): Porcentagem de tempo da CPU gasto no kernel (sistema). Se o sistema estiver movendo dados entre a memória e o swap, isso pode refletir no tempo do sistema.
